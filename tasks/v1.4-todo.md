**Description:** Evolve the core type system from a single `BehaviorTree` to a multi-tree `BTDocument`. Add the `SubTree` node kind. This is types-only — no runtime behavior changes yet.

### T5 — Validation rules R9, R10

**Description:** Add two new validation rules for SubTree nodes: R9 (treeRef must reference an existing tree in the document) and R10 (no circular subtree references).

**Acceptance criteria:**
- [ ] R9: Every `SubTree` node's `treeRef` must match the `name` of a tree definition in the document
  - Severity: error
  - Message: `SubTree node "${name}" references unknown tree "${treeRef}"`
- [ ] R10: No circular subtree references (tree A refs tree B which refs tree A)
  - Severity: error
  - Detect via DFS on the tree-reference graph
  - Message: `Circular subtree reference detected: ${cycle path}`
- [ ] Both rules operate on the full `BTDocument`, not a single tree
- [ ] Existing rules R1–R8 continue to operate per-tree within the document
- [ ] `validate()` signature updated to accept `BTDocument`

**Verification:**
- [ ] Unit tests:
  - R9: SubTree with valid treeRef → no issue
  - R9: SubTree with invalid treeRef → error
  - R10: No cycles → no issue
  - R10: A→B→A cycle → error with path
  - R10: A→B→C→A cycle → error with path
- [ ] Existing R1–R8 tests still pass

**Dependencies:** T1 (needs SubTree kind and document model)

**Files likely touched:**
- `src/core/validation/rules.ts`
- `src/core/validation/index.ts`
- `src/core/validation/types.ts` (new RuleIds)
- `tests/unit/core/validation/rules.test.ts`

**Estimated scope:** M

### T11 — Tree CRUD (create, rename, delete)

**Description:** Add create, rename, and delete operations for tree definitions within a document.

**Acceptance criteria:**
- [ ] **Create:** "+" button in tab bar creates a new tree definition with a Root node and auto-generated name ("Tree 2", "Tree 3", etc.). New tab becomes active.
- [ ] **Rename:** Double-click tab name → inline edit (same pattern as file rename T5 in v1.1). Enter confirms, Escape cancels. Renaming a tree updates all SubTree nodes that reference it by the old name.
- [ ] **Delete:** Right-click tab → context menu with "Delete" option (or "x" button on tab). Main tree cannot be deleted. Confirmation dialog: "Delete tree '{name}'? SubTree references will become invalid."
- [ ] After deletion, if the deleted tree was active, switch to the main tree
- [ ] Delete is a single undo step; undo restores the tree and all SubTree references

**Verification:**
- [ ] Manual browser test: full create → rename → delete cycle
- [ ] Renaming updates SubTree references in other trees
- [ ] Cannot delete the main tree
- [ ] Undo after delete restores everything
- [ ] Component test: create/rename/delete callbacks

**Dependencies:** T9 (needs tab UI for user interaction)
