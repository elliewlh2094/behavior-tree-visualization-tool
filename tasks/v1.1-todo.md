## Phase 2: UI Components

> **Phase 2 retrospective (added 2026-04-26 after Phase 2 shipped):** The plan called T4, T6, T9 "parallel," but T4 and T9 both edit `Toolbar.tsx`. Phase 2 was implemented serially in the order T6 → T4 → T9 (T6 first because it's fully isolated, then T4's filename UI, then T9's hook refactor on top). This avoided merge friction and let the hook contract reflect the final UI. The plan also deferred all e2e changes to T11 (Phase 4), but T9's start screen broke the existing `e2e/authoring.spec.ts` `beforeEach`. The minimal fix (click "New Tree" in `beforeEach`, rename the now-contradictory `'no splash or login'` test) shipped inside T9's commit so `main` did not regress. A dedicated `e2e/start-screen.spec.ts` is still T11's responsibility.
