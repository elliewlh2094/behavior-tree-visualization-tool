### T2 — Tree layout engine (pure function)

**Description:** Implement a custom recursive tree layout algorithm as a pure function. No dagre/elkjs dependency — BT is a strict hierarchy, not a general DAG. The function computes new positions for all nodes given a `BehaviorTree`, snapping to the grid.

**Acceptance criteria:**
- [x] New file `src/core/layout/tree-layout.ts` exports `computeTreeLayout(tree, options): Map<string, {x, y}>` with `LayoutOptions` interface
- [x] Root centered over its children; children below, siblings left-to-right by connection `order` value
- [x] All returned positions are snapped to `gridSize`
- [x] Orphaned nodes (no parent, not Root) placed in a column to the right of the main tree
- [x] Connection `order` values are NOT modified — covered by input-immutability test in T5
- [x] Function is pure: no store access, no side effects
- [x] ~70 lines implementation (tighter than estimated)

**Verification:**
- [x] `npm run typecheck` green
- [x] 100% coverage on `tree-layout.ts` from T5

**Dependencies:** None

**Files likely touched:**
- `src/core/layout/tree-layout.ts` (new)

## Phase 2: UI Integration

### T3 — Grid toggle button + Canvas integration

**Description:** Add a grid toggle to the toolbar and wire it to control overlay rendering in `Canvas.tsx`. Grid on → grid lines + full axes through world (0,0). Grid off → small ~50 px cross at world (0,0). Snap-to-grid stays active either way.

**Acceptance criteria:**
- [x] Toggle switch in toolbar — labeled `Grid` + welcome-ui-style switch (sky-700 track on, slate-300 off, white thumb slides). Replaces the originally-planned icon button per user feedback. See `memory/feedback_toggle_over_icon.md`.
- [x] Toggle calls `toggleGrid()` from store
- [x] Canvas reads `showGrid` from store:
  - `true`: renders `<Background variant="lines" gap={GRID_SIZE} />` plus `<AxisOverlay />` (full-screen pale-gray axes through world (0,0))
  - `false`: hides both; renders `<OriginCross />` instead (50 px slate-300 cross at world origin)
- [x] `AxisOverlay` (full axes) and `OriginCross` (short marker) are sibling components in `Canvas.tsx`. Single conditional at the canvas root chooses which to render.
- [x] Snap-to-grid behavior is unaffected (independent of overlay state)
- [x] Session-only: grid state resets on page refresh (persistence deferred to v1.3)

**Verification:**
- [x] Manual browser test 2026-04-26: toggle flips on/off, grid + axes appear/disappear together, short cross marks origin when off, snap unaffected, no regression.
- Skipped component test — Canvas overlay rendering is hard to assert through React Flow's internals; manual browser test + e2e (T6) cover it.

**Dependencies:** T1 (needs `showGrid` and `toggleGrid` in store)

### T4 — Auto layout button + store integration

**Description:** Add a "Layout" button to the toolbar that triggers the tree layout engine and applies the computed positions to all nodes as a single undo step. After applying, fit the canvas to the new layout.

**Acceptance criteria:**
- [ ] "Layout" button in toolbar (icon: layout/organize icon)
- [ ] Clicking the button:
  1. Calls `computeTreeLayout(tree, options)` with grid = 25, nodeWidth = 150, nodeHeight = 75, reasonable gapX/gapY
  2. Applies the resulting `Map<string, {x, y}>` to all node positions in one store mutation
  3. Creates a single undo snapshot (so the entire layout is one undo step)
  4. Calls `fitView()` on the ReactFlow instance after positions update
- [ ] Connection `order` values are NOT changed
- [ ] Works correctly with orphaned nodes (placed to the right)
- [ ] Works with empty tree (single Root) — no-op or trivial placement

**Verification:**
- [ ] Manual browser test: create a messy tree → click Layout → nodes reorganize in a clean top-down hierarchy
- [ ] Undo restores previous positions in one step
- [ ] Redo re-applies the layout
- [ ] Orphaned nodes appear to the right of the main tree
- [ ] Canvas zooms/pans to fit the new layout

**Dependencies:** T2 (needs `computeTreeLayout`)

**Files likely touched:**
- `src/components/toolbar/Toolbar.tsx`
- `src/store/bt-store.ts` (new `applyLayout` action)
- `src/components/canvas/Canvas.tsx` (fitView after layout)
