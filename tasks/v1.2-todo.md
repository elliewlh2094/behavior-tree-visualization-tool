### T2 — Tree layout engine (pure function)

**Description:** Implement a custom recursive tree layout algorithm as a pure function. No dagre/elkjs dependency — BT is a strict hierarchy, not a general DAG. The function computes new positions for all nodes given a `BehaviorTree`, snapping to the grid.

**Acceptance criteria:**
- [ ] New file `src/core/layout/tree-layout.ts` exports:
  ```ts
  export function computeTreeLayout(
    tree: BehaviorTree,
    options: { gridSize: number; nodeWidth: number; nodeHeight: number; gapX: number; gapY: number }
  ): Map<string, { x: number; y: number }>;
  ```
- [ ] Root placed at top center; children below, siblings left-to-right by connection `order` value
- [ ] All returned positions are snapped to `gridSize` (25 px)
- [ ] Orphaned nodes (no parent, not Root) placed in a row to the right of the main tree
- [ ] Connection `order` values are NOT modified — this is a position-only operation
- [ ] Function is pure: no store access, no side effects
- [ ] ~100 lines implementation

**Verification:**
- [ ] `npm run typecheck` green
- [ ] Function can be called standalone with a mock tree (tested in T5)

**Dependencies:** None

**Files likely touched:**
- `src/core/layout/tree-layout.ts` (new)

**Estimated scope:** M

### T4 — Auto layout button + store integration

**Description:** Add a "Layout" button to the toolbar that triggers the tree layout engine and applies the computed positions to all nodes as a single undo step. After applying, fit the canvas to the new layout.

**Acceptance criteria:**
- [ ] "Layout" button in toolbar (icon: layout/organize icon)
- [ ] Clicking the button:
  1. Calls `computeTreeLayout(tree, options)` with grid = 25, nodeWidth = 150, nodeHeight = 75, reasonable gapX/gapY
  2. Applies the resulting `Map<string, {x, y}>` to all node positions in one store mutation
  3. Creates a single undo snapshot (so the entire layout is one undo step)
  4. Calls `fitView()` on the ReactFlow instance after positions update
- [ ] Connection `order` values are NOT changed
- [ ] Works correctly with orphaned nodes (placed to the right)
- [ ] Works with empty tree (single Root) — no-op or trivial placement

**Verification:**
- [ ] Manual browser test: create a messy tree → click Layout → nodes reorganize in a clean top-down hierarchy
- [ ] Undo restores previous positions in one step
- [ ] Redo re-applies the layout
- [ ] Orphaned nodes appear to the right of the main tree
- [ ] Canvas zooms/pans to fit the new layout

**Dependencies:** T2 (needs `computeTreeLayout`)

**Files likely touched:**
- `src/components/toolbar/Toolbar.tsx`
- `src/store/bt-store.ts` (new `applyLayout` action)
- `src/components/canvas/Canvas.tsx` (fitView after layout)

**Estimated scope:** S–M
